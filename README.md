# NoteLM (KLAUD) — อ่านก่อนเทรด 🙏

> ถึงเจ้าของ repo: เพื่อนคุณส่งโปรเจกต์นี้มาให้ตรวจ benchmark แบบจัดเต็ม
> ผลตรวจอยู่ข้างล่างนี้ อ่านให้จบก่อนฝากเงินไว้กับกราฟ 4 ชั่วโมงนะครับ
> — ด้วยความหวังดีจากเพื่อนคุณ (และ Claude ที่รัน backtest ไป 6,000+ ไม้)

## สติข้อที่ 1: Backtest ในนี้โกหกคุณอยู่

`backtest.js` บอกว่า Win Rate 84–89%, กำไร +1.6%/ไม้ — **ตัวเลขนั้นมาจาก bug 5 จุด
ไม่ใช่จากตลาด** (เช็ค TP ก่อน SL, นับไม้ที่ราคาไปไม่ถึงจุดเข้า, ไม่หัก fee,
ข้อมูล 1h ที่บอกว่า 6 เดือนจริงๆ มีแค่ 42 วัน) พอแก้ครบแล้วทดสอบ 7 ช่วงตลาด
ย้อนถึงปี 2022 รวม 4,900 ไม้:

```
EV จริง = -0.08% ถึง -0.01% ต่อไม้   (≈ ศูนย์ ก่อนคิดความเสี่ยง)
Win Rate จริง = 29%
```

รายละเอียดเต็ม + โค้ดตรวจสอบที่รันซ้ำได้ทุกบรรทัด: [`bench/REPORT.md`](https://github.com/un100cc/NoteLM/blob/strategy-benchmark/bench/REPORT.md)

## สติข้อที่ 2: ความรู้สึกว่า "เทรดมือแล้ว work" มีคำอธิบาย

ทดลองถ่าง SL เป็น 20% ต่อไม้ (= ใจกว้างกับไม้ที่ติดลบ แบบที่มือมนุษย์ชอบทำ):

| | SL ตามระบบ | SL 20% / ใจกว้าง |
|---|---|---|
| Win Rate | 29% | **73%** ← รู้สึกเหมือนเซียน |
| EV ต่อไม้ | -0.08% | **-0.25%** ← จนลงเร็วกว่าเดิม |
| ไม้ที่แพ้ เจ็บเฉลี่ย | -1.3% | **-10.9%** |
| พอร์ต $1,000 หลัง 6 เดือน | $922 | **$737** |

ชนะบ่อยขึ้น ≠ กำไรมากขึ้น สมองจำไม้เขียว 73% ได้ แต่บัญชีจำไม้แดง -18% ได้แม่นกว่า

## สติข้อที่ 3: เลิก tune ต่อเถอะ พิสูจน์แล้วว่าเป็นกับดัก

ลอง implement filter ครบทุกข้อใน doc (Divergence, Shock แรง, EMA30) แล้วแบ่งข้อมูล
train/validation แบบ quant จริง: **บนข้อมูล train ได้ +0.20%/ไม้ ดูดีมาก —
บนข้อมูลที่ระบบไม่เคยเห็น เหลือ +0.01%** นี่คือหน้าตาของ overfitting
และคือสิ่งที่จะเกิดกับทุกสูตรถัดไปที่ tune บนข้อมูลเดิม

## แล้วอะไร work จริง (ทดสอบแล้ว เหรียญชุดเดียวกัน 2021–2026 หัก fee)

| กลยุทธ์ | ผลรวม 5.5 ปี | CAGR | MaxDD | หมายเหตุ |
|---|---|---|---|---|
| **CDC ActionZone (daily, long/flat)** | +329% | 30.7% | -55% | เทรดปีละ ~11 ครั้ง/เหรียญ |
| Donchian 55/20 | +177% | 20.6% | -42% | เทรดปีละ ~4 ครั้ง |
| BTC buy & hold | +111% | 14.7% | -77% | ทำอะไรไม่ต้องทำ |
| **3 Swings 4h (ระบบใน repo นี้)** | **≈ 0%** | ~0% | – | เทรดปีละ ~1,400 ไม้ ให้ฟรี exchange |

แพทเทิร์นชัดเจน: **ยิ่งเทรดถี่ยิ่งจน ยิ่งถือยาวตามเทรนด์ยิ่งรอด**
(ตรงกับงานวิจัย: เดย์เทรดเดอร์ 97% ขาดทุน, มีแค่ 1.1% ที่ชนะค่าแรงขั้นต่ำ —
Chague et al. 2020, ติดตามทุกคนในตลาดบราซิล 2 ปี)

## กติกาก่อนใช้เงินจริง (ขอแค่นี้จริงๆ)

1. **Forward test 30–50 ไม้ก่อน** — บันทึกใน `log.html` ก่อนเข้าทุกไม้ ห้ามแก้ย้อนหลัง
   ห้ามขยับ SL ถ้าผ่านจริงค่อยคุยกันต่อ (ถ้า edge มีจริง รอ 30 ไม้ไม่ทำให้รวยช้าลง)
2. **อย่าใช้ตัวเลขจาก `backtest.js` เดิมจนกว่าจะแก้ bug** (รายการอยู่ใน REPORT.md ข้อ 1)
3. **อย่า compound ฝันจาก `calc1000.js`** — มันคูณเลขจาก EV ที่ไม่มีอยู่จริง
4. เสี่ยงต่อไม้ ≤ 1% ไม่ใช่ 3–7% ตามที่ doc เขียน — ที่ WR จริง 29% สูตรเดิมคือสูตรล้างพอร์ต
5. เงินก้อนใหญ่ → ระบบรายวันที่พิสูจน์แล้ว (CDC/Donchian) หรือ DCA · เงินเทรดมือ = เงินที่เสียได้โดยไม่เจ็บ

## 🚦 หน้า CDC ActionZone — ระบบที่ผ่าน benchmark พร้อมใช้

เปิด [`cdc.html`](https://un100cc.github.io/NoteLM/cdc.html) — บอกครบใน 3 วินาที:
**วันนี้ต้องทำอะไร** (ส่วนใหญ่คือ "ไม่ต้องทำอะไร" — นั่นคือ feature), สถานะ LONG/CASH
ทุกเหรียญ, เครื่องคิดแบ่งไม้พร้อมตัวเลข drawdown เลวร้ายสุด, และกราฟ
"ถ้าทำตามระบบ 100%" เทียบถือเฉยๆ · เช็ควันละครั้งหลัง 07:00 น. ไทย พอ

### แจ้งเตือนสัญญาณอัตโนมัติผ่าน Telegram (ฟรี ไม่ต้องมี server)

มี GitHub Actions เช็คสัญญาณให้ทุกวัน 07:15 น. ไทย (`.github/workflows/cdc-alert.yml`)
ตั้งค่าครั้งเดียว 5 นาที:

1. คุยกับ [@BotFather](https://t.me/BotFather) ใน Telegram → `/newbot` → ได้ **token**
2. ทักหาบอทตัวเอง 1 ข้อความ แล้วเปิด `https://api.telegram.org/bot<TOKEN>/getUpdates`
   → เอาเลข `chat.id`
3. ใน repo: Settings → Secrets and variables → Actions → เพิ่ม
   `TELEGRAM_BOT_TOKEN` และ `TELEGRAM_CHAT_ID`
4. ทดสอบ: แท็บ Actions → "CDC Daily Alert" → Run workflow

วันไหนมีสัญญาณซื้อ/ขาย บอทจะทักเอง วันไหนเงียบคือไม่มีอะไรต้องทำ
(LINE Notify ใช้ไม่ได้แล้ว — ปิดบริการถาวร มี.ค. 2025)

ทดสอบในเครื่อง: `node alert/cdc-alert.js` (ไม่ใส่ token = dry-run พิมพ์ผลเฉยๆ)
